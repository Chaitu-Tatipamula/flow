Build artifacts

One of the main features of the flow testing framework is the production of rich execution reports.

Until such a time as upload-artifact#14 is addressed, we're reduced to abusing github pages to show these artifacts to best effect. See test.yml, mutation.yml and regen_index.pl for the gory details.

Execution reports

These reports are the result of comparing a unified model of system behaviour against:

  • an instance of the complete system (The "app-itest" report)
  • system components in isolation (everything else).
